## 개발 환경 세팅

### 1. JDK 21 설치

#### Windows

**다운로드 및 설치**
1. [Oracle JDK 다운로드](https://www.oracle.com/java/technologies/downloads/#java21)
2. Windows x64 Installer 다운로드
3. 설치 실행 (기본 경로: `C:\Program Files\Java\jdk-21`)

**환경 변수 설정**
1. Windows 검색에서 "환경 변수" 입력
2. "시스템 환경 변수 편집" 클릭
3. 환경 변수 버튼 클릭
4. 시스템 변수에서 "새로 만들기"
   - 변수 이름: `JAVA_HOME`
   - 변수 값: `C:\Program Files\Java\jdk-21`
5. 시스템 변수에서 `Path` 선택 후 편집
   - "새로 만들기" → `%JAVA_HOME%\bin` 입력

**설치 확인**
```cmd
java -version
javac -version
```

#### macOS

```bash
brew install openjdk@21
sudo ln -sfn /opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

**설치 확인**

### 5. GitHub 계정 및 토큰(PAT) 설정

GitHub는 이제 **계정 비밀번호로 Git 인증을 허용하지 않고**,  
반드시 **토큰(Personal Access Token, PAT)** 을 사용해야 합니다.

우리는 SSH 대신 **HTTPS + 토큰 방식**으로 설정합니다.

#### 1) GitHub 계정 생성

1. https://github.com 접속
2. `Sign up` 버튼 클릭
3. 이메일 / 비밀번호 / 사용자 이름 입력
4. 메일 인증 완료

#### 2) Personal Access Token 생성 (Developer settings)

1. GitHub 로그인
2. 오른쪽 상단 프로필 아이콘 클릭 → `Settings`
3. 왼쪽 메뉴 맨 아래 `Developer settings` 클릭
4. `Personal access tokens` → `Tokens (classic)` 선택  
   (또는 Fine-grained token 사용 가능)
5. `Generate new token (classic)` 클릭
6. 이름: 예) `java-practice-token`
7. 만료 기간: 적절히 선택 (예: 90일, 6개월 등)
8. 권한(Scopes): 최소한 아래 체크
   - `repo` (저장소 접근)
9. 맨 아래 `Generate token` 클릭
10. 생성된 토큰 문자열을 **안전한 곳에 복사/저장**  
    (한 번만 보여주므로 꼭 복사해 두기)

> 토큰은 **비밀번호와 동일한 수준으로 중요**합니다.  
> 노출되면 바로 GitHub에서 Revoke(무효화)하고 새로 발급해야 합니다.

#### 3) 토큰으로 Git 인증하기

- Git을 통해 **HTTPS URL**로 `clone`, `push` 등을 할 때  
  처음 한 번 GitHub 인증을 요청합니다.
- 이때:
  - Username: GitHub 아이디
  - Password: 방금 만든 **Personal Access Token** 입력
- 한 번 입력하면 OS의 자격 증명 관리자(credential manager)에 저장되어  
  이후에는 자동으로 인증되는 경우가 많습니다.

> 참고: 나중에 토큰을 잊어버리면 Developer settings에서 다시 새 토큰을 발급하면 됩니다.

## Java 기초

### 1. 변수 (Variable)

#### 변수가 필요한 이유

다음 코드를 보겠습니다:

```java
System.out.println(10);
System.out.println(10);
System.out.println(10);
```

여기서 숫자 10을 20으로 바꾸려면 3곳을 모두 수정해야 합니다. 만약 100곳이라면?

변수는 이런 문제를 해결합니다.

```java
int a = 10;
System.out.println(a);
System.out.println(a);
System.out.println(a);
```

이제 `a`의 값만 바꾸면 모든 출력이 변경됩니다.

---

#### 변수 선언과 초기화

**선언**: 변수를 만드는 것
```java
int a;  // 정수를 저장할 공간 생성
```

**초기화**: 처음으로 값을 저장하는 것
```java
a = 10;  // 변수 a에 10을 저장
```

**선언과 초기화를 동시에**
```java
int a = 10;
```

**여러 변수 선언**
```java
int a, b;
int c = 3, d = 4;
```

---

#### 변수 타입

**정수형**
```java
int age = 25;           // 정수 (약 20억)
long population = 10000000000L;  // 큰 정수 (L 붙임)
```

**실수형**
```java
double price = 10.5;    // 실수 (소수점)
```

**문자형**
```java
char grade = 'A';       // 문자 하나 (작은따옴표)
String name = "홍길동"; // 문자열 (큰따옴표)
```

**논리형**
```java
boolean isActive = true;  // true 또는 false
```

---

#### 실무에서 자주 쓰는 타입

- **정수**: int, long
- **실수**: double
- **문자열**: String
- **논리**: boolean

나머지는 거의 사용하지 않습니다.

---

#### 리터럴 (Literal)

코드에 직접 입력한 고정된 값을 리터럴이라고 합니다.

```java
int a = 100;        // 100은 정수 리터럴
double b = 10.5;    // 10.5는 실수 리터럴
String c = "Hello"; // "Hello"는 문자열 리터럴
```

---

#### 변수 명명 규칙

**규칙 (필수)**
- 숫자로 시작 불가: `1num` (X), `num1` (O)
- 공백 불가
- 예약어 사용 불가: `int`, `class`, `public` 등
- 영문자, 숫자, $, _ 만 사용

**관례 (권장)**
- 소문자로 시작
- camelCase 사용: `orderDetail`, `userName`
- 의미있는 이름: `a`, `b` (X) → `studentCount`, `maxScore` (O)

### 2. 연산자

#### 산술 연산자

```java
int a = 10;
int b = 3;

System.out.println(a + b);  // 13
System.out.println(a - b);  // 7
System.out.println(a * b);  // 30
System.out.println(a / b);  // 3 (정수 나눗셈)
System.out.println(a % b);  // 1 (나머지)
```

#### 비교 연산자

```java
int a = 10;
int b = 20;

System.out.println(a == b);  // false (같다)
System.out.println(a != b);  // true (다르다)
System.out.println(a > b);   // false
System.out.println(a < b);   // true
System.out.println(a >= b);  // false
System.out.println(a <= b);  // true
```

#### 논리 연산자

```java
boolean a = true;
boolean b = false;

System.out.println(a && b);  // false (AND)
System.out.println(a || b);  // true (OR)
System.out.println(!a);      // false (NOT)
```

### 6. 메서드

```java
public class Calculator {
```

### 2. 클래스와 객체

#### 클래스: 설계도

클래스는 객체를 만들기 위한 틀입니다.

```java
public class Car {
    // 속성 (필드)
    String color;
    int speed;

    // 기능 (메서드)
    void accelerate() {
        speed += 10;
    }

    void brake() {
        speed -= 10;
    }
}
```

#### 객체: 실체

```java
// 객체 생성
Car myCar = new Car();
myCar.color = "red";
myCar.speed = 0;

// 메서드 호출
myCar.accelerate();
System.out.println("속도: " + myCar.speed);  // 10
```

**비유**
- 클래스: 붕어빵 틀
- 객체: 실제 붕어빵

### 3. 생성자 (Constructor)

객체를 생성할 때 초기화하는 특별한 메서드입니다.


**사용**
```java
Person person1 = new Person("홍길동", 25);

```

### 4. 캡슐화 (Encapsulation)

데이터를 보호하고 외부 접근을 제어하는 것입니다.

#### private와 getter/setter

```java
public class Account {
    private int balance;  // 외부에서 직접 접근 불가

    // Getter: 값을 읽기
    public int getBalance() {
        return balance;
    }

    // Setter: 값을 설정 (검증 가능)
    public void setBalance(int balance) {
        if (balance >= 0) {
            this.balance = balance;
        } else {
            System.out.println("잔액은 0 이상이어야 합니다.");
        }
    }
}
```

**왜 캡슐화를 하나?**
1. 데이터 보호
2. 유효성 검증 가능
3. 내부 구현 변경 시 외부 영향 최소화

### 5. 상속 (Inheritance)

기존 클래스를 확장하여 새로운 클래스를 만드는 것입니다.

```java
// 부모 클래스
public class Animal {
    String name;

    public void eat() {
        System.out.println(name + "이(가) 먹습니다.");
    }
}

// 자식 클래스
public class Dog extends Animal {
    public void bark() {
        System.out.println(name + "이(가) 짖습니다: 멍멍!");
    }
}
```

**사용**
```java
Dog dog = new Dog();
dog.name = "바둑이";
dog.eat();   // 상속받은 메서드
dog.bark();  // 자신의 메서드
```

### 6. 다형성 (Polymorphism)

같은 타입이지만 다른 동작을 하는 것입니다.

#### 메서드 오버라이딩

```java
public class Animal {
    public void makeSound() {
        System.out.println("동물 소리");
    }
}

public class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("멍멍!");
    }
}

public class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("야옹!");
    }
}
```

**사용**
```java
Animal animal1 = new Dog();
Animal animal2 = new Cat();

animal1.makeSound();  // 멍멍!
animal2.makeSound();  // 야옹!
```

### 7. 패키지와 접근 제어자

```java
// com/example/study/model/Person.java
package com.example.study.model;

public class Person {
    private String name;
}
```

```java
// com/example/study/Main.java
package com.example.study;

import com.example.study.model.Person;

public class Main {
    public static void main(String[] args) {
        Person person = new Person();
    }
}
```

#### 접근 제어자

| 제어자 | 클래스 내부 | 같은 패키지 | 하위 클래스 | 모든 곳 |
|--------|------------|------------|------------|---------|
| private | O | X | X | X |
| default | O | O | X | X |
| protected | O | O | O | X |
| public | O | O | O | O |

## Git/GitHub 워크플로

### 1. Git 기본 개념

**주요 용어**
- Repository: 프로젝트 저장소
- Commit: 변경사항 기록
- Branch: 독립적인 작업 공간
- Merge: 브랜치 병합
- Remote: 원격 저장소

---

### 2. Git 기본 명령어

#### 저장소 초기화

```bash
# 새 저장소 생성
mkdir my-project
cd my-project
git init

# 원격 저장소 복제
git clone https://github.com/username/repository.git
```

#### 기본 워크플로

```bash
# 상태 확인
git status

# 파일 스테이징
git add filename.txt
git add .

# 커밋 생성
git commit -m "커밋 메시지"

# 원격 저장소에 푸시
git push origin main

# 원격 저장소에서 가져오기
git pull origin main
```

---

### 3. 브랜치 관리

```bash
# 브랜치 목록
git branch

# 브랜치 생성
git branch feature/login

# 브랜치 전환
git checkout feature/login
git switch feature/login

# 브랜치 생성 및 전환
git checkout -b feature/signup
git switch -c feature/signup

# 브랜치 병합
git checkout main
git merge feature/login

# 브랜치 삭제
git branch -d feature/login
```

---

### 4. GitHub 저장소 연결

```bash
# 원격 저장소 추가
git remote add origin git@github.com:username/repository.git

# 확인
git remote -v

# 첫 푸시
git push -u origin main
```

---

### 5. Pull Request 워크플로

```bash
# 1. 최신 코드 받기
git checkout main
git pull origin main

# 2. 새 브랜치 생성
git checkout -b feature/add-function

# 3. 코드 작성 및 커밋
git add .
git commit -m "feat: 새로운 기능 추가"

# 4. 원격 저장소에 푸시
git push origin feature/add-function
```

**GitHub에서 PR 생성**
1. GitHub 저장소 페이지 접속
2. "Compare & pull request" 클릭
3. PR 제목과 설명 작성
4. "Create pull request" 클릭
5. 리뷰 후 "Merge pull request"

### 과제 2: 계산기 클래스

**목표**: 클래스와 메서드 이해

Calculator 클래스를 만드세요.

**요구사항**
- add(int a, int b): 덧셈
- subtract(int a, int b): 뺄셈
- multiply(int a, int b): 곱셈
- divide(int a, int b): 나눗셈

**사용 예시**
```java
Calculator calc = new Calculator();
System.out.println(calc.add(10, 5));      // 15
System.out.println(calc.subtract(10, 5)); // 5
```
